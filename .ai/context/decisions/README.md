# Architecture Decision Records (ADRs)

One file per architectural decision. Lets AI (and humans) understand **why** we chose what we did — so we don't re-litigate it.

## When to write an ADR

- Picking a major dependency
- Choosing between two valid architectures
- Reversing a previous decision
- Anything you'd defend in a code review

## Format

Copy `0000-template.md`, increment the number, fill it in. Commit.
