# Database Schema

Snapshot of tables and relationships. Update when migrations change.

## Conventions

- **IDs:** `uuid` primary keys, generated by DB default (`gen_random_uuid()`)
- **Timestamps:** `created_at`, `updated_at` (both `timestamptz` with defaults)
- **Soft delete:** `deleted_at timestamptz nullable` where applicable — never hard-delete user data
- **Naming:** `snake_case` columns, `plural` table names
- **Foreign keys:** `<table>_id` with explicit `references` and `on delete` rule
